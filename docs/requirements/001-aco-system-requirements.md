## 🧱 기능 상세 정의

### 1️⃣ 대시보드 (Dashboard)

**요구사항**
- 전체 구성원 현황 및 통계 정보를 한눈에 확인
- 주요 지표 및 요약 정보 제공

**세부 기능**
1. 전체 구성원 수 표시
2. 악기별 구성원 수 통계
3. 파트별 구성원 분포
4. 이번 주 연습일정 요약
5. 출석률 통계 (향후 확장)

**UI 구성**
- 카드 형태의 통계 위젯
- 차트/그래프를 통한 시각화 (선택사항)
- 최근 활동 내역 (선택사항)

---

### 2️⃣ 회원 관리 시스템 (Member Management)

**요구사항**
- 구성원 정보 등록, 수정, 삭제, 조회
- 악기 및 파트 정보 관리
- 구성원 목록 필터링 및 검색

**세부 기능**

#### 2-1. 회원 목록 조회
- 전체 회원 목록 표시 (테이블 형태)
- 컬럼: No., 이름, 악기, 파트, 비고
- 정렬 기능 (이름, 악기, 파트 기준)
- 검색 기능 (이름, 악기로 검색)

#### 2-2. 회원 등록
- 이름 입력
- 악기 선택 (드롭다운)
- 파트 선택 (1st, 2nd, 3rd 등 또는 빈 값)
- 비고 입력 (선택사항)

#### 2-3. 회원 수정
- 기존 정보 수정
- 악기/파트 변경 가능

#### 2-4. 회원 삭제
- 회원 정보 삭제 (확인 다이얼로그)

#### 2-5. 악기 정보 관리
- 악기 목록 관리 (플룻, 오보에, 클라리넷, 트럼펫, 호른, 튜바 등)
- 악기별 한글명, 영어명, 약식 저장
- 악기 추가/수정 기능 (향후 확장)

**데이터 구조**
```json
{
  "members": [
    {
      "id": 1,
      "name": "김민정",
      "instrument": "Fl",
      "part": "1st",
      "remarks": ""
    }
  ],
  "instruments": [
    {
      "id": 1,
      "name": "플룻",
      "english": "Flute",
      "abbreviation": "Fl"
    }
  ]
}
```

### 3️⃣ 연습일정 관리 시스템 (Practice Schedule Management)

**요구사항**
- 요일별 연습일정 설정 및 관리
- 구성원별 출석 가능 일정 등록
- 캘린더 뷰를 통한 일정 시각화
- 일정 현황 통계 및 요약

**세부 기능**

#### 3-1. 일정 현황 조회
- 주간 일정 현황 표시 (월~일)
- 구성원별 출석 가능 요일 표시
- 요일별 출석 가능 인원 수 표시
- 시각적 표시 (체크박스 또는 아이콘)

#### 3-2. 캘린더 뷰
- 월간/주간 캘린더 뷰 제공
- 각 날짜에 출석 가능 인원 수 표시
- 날짜 클릭 시 해당 날짜 출석 가능 구성원 상세 정보 표시
- 이전/다음 월/주 네비게이션
- 오늘 날짜 하이라이트

#### 3-3. 일정 등록/수정
- 구성원별 출석 가능 요일 선택
- 주간 단위로 일정 관리
- 일정 수정 기능

#### 3-4. 일정 통계
- 요일별 출석 가능 인원 수
- 구성원별 주간 출석 가능 일수
- 통계 요약 정보

**데이터 구조**
```json
{
  "schedules": [
    {
      "memberId": 1,
      "memberName": "한동윤",
      "availableDays": ["목", "금", "토"]
    }
  ],
  "weeklyStats": {
    "월": 5,
    "화": 0,
    "수": 3,
    "목": 2,
    "금": 4,
    "토": 2,
    "일": 3
  }
}
```

## 🔧 기술 고려사항

1. **프레임워크**
   - Next.js (App Router 권장)
   - React 18+
   - TypeScript

2. **데이터 저장**
   - 초기: JSON 파일 기반 (로컬 또는 public 폴더)
   - 파일 구조: `/data/members.json`, `/data/instruments.json`, `/data/schedules.json`
   - 향후: 데이터베이스 연동 (Firebase, PostgreSQL 등)

3. **UI/UX**
   - 반응형 디자인
   - 직관적인 테이블 및 폼 구성
   - 모던한 UI 라이브러리 활용 (shadcn/ui, Tailwind CSS 등)

4. **상태 관리**
   - React Context 또는 Zustand (선택사항)
   - 서버 상태는 JSON 파일 읽기/쓰기로 처리
